Commission Transactions Management System


Commission processing is a time-consuming and difficult operation for any multi-broker selling company or broker aggregator. Keeping track with several brokers and lenders with varying commission arrangements can be challenging. Processing and transmitting fee payments to various recipients, as well as manipulating and validating data from various lender archives, is time-consuming, costly, and vulnerable to error. You need a solution that takes care of it automatically, adapts to the changing market needs, and does not need a large investment in labor.


Project Overview


  1. The Digital Commission Management System is a two-pronged real estate management approach for brokers and agents that incorporates technology (Software as a Service or SaaS) and people (Commissions as a Service or CaaS). Listings, leads, reviews, emails, commissions, billing, filing, and more are all managed across the website.

  2. From any location with an internet connection, users can view all purchases, payables, agent billing, fees, and the general ledger. Access protocols for geographic and individual employees, as well as multi-office management reports, make it possible to handle multi-office businesses. Users can bill agents, print commission checks, and create letters from several models, and commissions and transaction-related posts to the general ledger are handled automatically.

  3. The commission system imports and validates loan data from various lender files/schedules, integrates with broker CRM to generate settlements data, provides a strong batch exception handling to acknowledge and manage loan exceptions from lender files, supports a virtually unlimited array of calculation models, processes multiple commission runs with only a few clicks, generates, and distributes commission reports.

  4. Architects advise on the development of an end-to-end platform/system that is designed for speed and size. That is the collection and delivery of fee payments in a detailed and effective manner. Data import and validation are also seamless.


Developed Solution


The end-to-end solution specifically targets the challenges experienced by aggregators. It is a two-pronged solution, comprised of technology (Software as a Service, or SaaS) and people (Commissions as a Service, or CaaS), which:

  1. Seamlessly imports and validates loan data from various lender files/schedules.

  2. Integrates with broker CRM to generate settlement data.

  3. Provides strong batch exception handling to acknowledge and manage loan exceptions from lender files.

  4. Supports a virtually unlimited array of calculation models.

  5. Processes numerous commissions runs with just a couple of clicks.

  6. Generates and distributes payment invoices to recipients via e-mail, broker portal, or shared network drive.
  7. Generates ABA-compatible payment files for bulk payment processing.
  8. Is cost-effective, comprehensive, and scalable.
  9. Offer a range of commission structure options, aimed solely at maximizing a broker’s profitability. We give brokers the choice of a competitive flat-fee model, a flexible percentage model, or a scalable transaction-based model. Our brokers are free to choose the commission model that suits them. They are also free to switch to another commission structure at any time, without penalty.


Commission Processing Screen


The Commission Processing Screen may be used to perform the following activities:

  1. Select commission detail records for payment.
  2. Review and manually edit the commission details.
  3. Place commission details records on hold.
  4. Search for and attach reservations to the agent for commission payment.
  5. Detach commission detail records from an agent or source.
  6. Pay commissions on selected commission details.
  7. Create a Transfer Report.
  8. Print a Commissions Payable report for the highlighted agent, or source in the top frame. This report shows held payments as well as AR payments.
  9. Place the selected guest's commission payment on hold to offset the agent's or source's accounts receivable account.
  10. Prepare files for EFT.


Key features


End-to-end solution is specifically designed for lenders and covers the entire process:

  1. Maintaining broker information & channel relationships

  2. Managing payment rules

  3. Importing loans data

  4. Reconciling, validating & matching information.

  5. Processing payment calculations

  6. Distributing invoices

  7. Integrating with core systems

  8. Full automation capability of transaction processing, commission holding, and distribution through local banking and automatic ABA file generation.

  9. Place the selected guest's commission payment on hold to offset the agent's or source's accounts receivable account.

  10. Prepare files for EFT.

  11. Processing payment queries


Technology Stack


Previous system was developed using PERL, JQUERY, MYSQL, RED HAT LINUX, SHELL SCRIPTING etc. But we developed this system using new technology like: Object Oriented Python, MariaDB, Linux Server/Cloud, AngularJS, Shell scripting, Triggers/stored procedures, and Advanced Web Technologies. Here is why we choose to use this stack and why is it better:

  1. In this system MariaDB was chosen over MYSQL because it offers more security feature like database encryption, LDAP and PAM authentication, security and password checks that makes the platform more secure, its faster and provides better user support than MYSQL

  2. Python was chosen over java and .net as it is dynamically typed, it has a lot of flexibility. Although Java has strict guidelines for creating functionality, Python has a variety of approaches to the same issue. The language is also more error-tolerant, making it easier to compile and execute your software.

  3. Two-way data binding, dependency injection, directives, and community support were some of the main features because AngularJS was chosen, also the MVC architecture of AngularJS that allows creation of dynamic and robust web applications. Splitting applications in MVC architecture is possible for AngularJS, which allows AngularJS mobile application development companies to save time. Experts can meet schedules and completing projects on schedule. Shell scripting was chosen over for its ease of use, features of automation of frequently used operations, system admin task automation, running sequence of commands as a single, and portability.
