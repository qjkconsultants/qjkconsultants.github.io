Python ETL Reporting Framework

Last Updated On:
January 1st, 2024
Author:
Qaisar Khan
Client:
ABN AMRO Clearing

Designed and develop new framework for the development of Regulatory Reporting projects, using component-based architecture, adhering strictly to the object-oriented design principles.

Aim is to allow developers focus on the business requirements and not utilize time on low level technical complexities, such as reading data from multiple sources, pre-processing/post processing of the data, generating multiple reports as outputs in dynamically created directories.

Component Based Architecture


The framework is developed using component-based architecture and developer could simply create a component for their business entity.

  1. The component allow access to multiple data sources as input (SQL Server/DB2/PostgreSQL/Excel).
  2. Each component has object-oriented classes (with base and abstract classes) to perform:
    1. Pre-processing of the raw data and rates conversion (Foreign Currency to SGD/HKD).
    2. Netting/aggregation/mathematical calculations using Pandas/NumPy.
    3. Map processed data frames to the template required by external source (Agile Reporter)
    4. Post-processing to apply additional rules on the mapped data and generate outputs.
    5. The multiple reports are then uploaded to Agile Reporter for Finance to further utilize.

Regulatory Reporting Framework Architecture Overview


  1. Regulatory Reporting Framework is designed to facilitate the seamless development of Regulatory Reporting projects.
  2. The overarching goal is to enable developers to concentrate on business requirements without being encumbered by low-level technical complexities.
  3. It employs a component-based architecture, ensuring modularity, reusability, and maintainability.
  4. Components, including data access, pre-processing and conversion, aggregation and calculations, data mapping, post-processing, and integration with Agile Reporter, contribute to an efficient, flexible, and accurate development process.

Advantages:

  1. Efficiency: Developers focus on business logic, reducing development time and effort.
  2. Flexibility: Supports various data sources, ensuring adaptability to diverse environments.
  3. Modularity: Components promote code reuse and maintainability.
  4. Data Accuracy: Systematic pre-processing and mapping enhance the accuracy of regulatory reports.

Product Develoment


Developed Reporting System for multiple regulatory authorities (Singapore/Hongkong), which:

  1. Retrieves the trading related transactions from multiple data sources.
  2. Separate components utilize the data and perform calculations and manipulations.
  3. Ledger balances are netted and mapped with GLC codes to segregate balances into different buckets of assets, liabilities, equity, etc. Profit & loss ratios are calculated.
  4. Multiple data frames mapped to the templates required, generating outputs, populating balances into external source (Agile Reporter) for MAS and HKMA regulatory reporting.
  5. ETL Solution Framework automates the entire process, guaranteeing 100% reconciliation and eliminating errors introduced by manual interventions.
  6. The solution is crafted using Object-Oriented Python, Pandas/Numpy for efficient data handling, and seamlessly integrates with various databases, including SQL Server, PostgreSQL, and IBM DB2.
  7. Developed Reconciliation Reports for the business components, so that business analysts could understand the rules implemented and supervise the transformation of the data to generate final reports.
  8. Implemented test driven development approach, creating unit tests for the functionality, integration tests for the output data, and code is deployed to multiple environments using Jenkins & CI/CD pipelines, configured the image to run in container via open shift.
  9. Developed an application to initiate the open shift job, creating a pod, running the business logic inside the container, capturing the logs of the running pods and update the status of jobs in the schedular up on completion. The processing logs are configured and made available for the users to view via SPLUNK.
  10. Automate the deployment process of applications developed (using Jenkins & CI/CD pipeline), configured the applications to run via container and OpenShift.
