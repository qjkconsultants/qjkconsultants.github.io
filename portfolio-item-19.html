Generative AI Solution using AWS Bedrock

Last Updated On:
January 1st, 2024
Author:
Qaisar Khan
Client:
ABN AMRO Clearing

Project Details

  1. Currently ABN AMRO is required to report on ESG data from our clients, which is being manually extracted currently from the annual reports (pdf).
  2. ESG stands for Environmental, Social and Governance. This is often called sustainability. In a business context, sustainability is about the companyâ€™s business model, i.e. how its products and services contribute to sustainable development.
  3. This is quite some manual labor, which the company would like to replace by utilizing Gen-AI and then doing verification rather than extracting the data themselves manually. Additionally, multiple other departments need to extract financial information from these annual reports, which could also be automated.

Implementation

  1. Designed/Develop solution using Gen-AI, to extract data using Amazon Bedrock, creating knowledge bases implementing AI technique - RAG.
  2. Retrieval-Augmented Generation (RAG) enhances the capabilities of a large language model (LLM) by allowing it to access and incorporate relevant information from external sources, like a knowledge base or database, before generating a response, essentially making its outputs more accurate and up to date by retrieving contextually relevant data in real-time instead of relying solely on its pre-trained knowledge.
  3. It is the process of optimizing the output of a large language model, so it references an authoritative knowledge base outside of its training data sources before generating a response.
  4. Knowledge base for Amazon Bedrock help you take advantage of Retrieval Augmented Generation to augment responses generated by Large Language Models (LLMs).
  5. When you set up a knowledge base with your data sources, your application can query the knowledge base to return information to answer the query either with direct quotations from sources or with natural responses generated from the query results.
  6. With knowledge bases, you can build applications that are enriched by the context that is received from querying a knowledge base. It enables a faster time to market by abstracting from the heavy lifting of building pipelines and providing you an out-of-the-box RAG solution to reduce the build time for your application.
  7. Adding knowledge base also increase cost-effectiveness by removing the need to continually train your model to be able to leverage your private data.
  8. Developed AI workflows connecting Amazon Bedrock prompts, knowledge base, and other AWS services to allow the application query the knowledge bases and extract the data.
  9. Implemented Lambda functions/API Gateway to submit request to the AI solution for data retrieval, generating ESG report, which is then uploaded to S3 bucket, available for download.
