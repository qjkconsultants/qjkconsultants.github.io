Generative AI Solution using AWS Bedrock

Last Updated On:
January 1st, 2024
Author:
Qaisar Khan
Client:
ABN AMRO Clearing

Project Overview

This project aimed to **automate the extraction, analysis, and reporting of ESG (Environmental, Social, Governance) data** from annual reports, which were previously processed manually. Manual extraction was slow, error-prone, and inconsistent. By leveraging **Generative AI and AWS Bedrock**, the solution introduced an **automated, scalable, and accurate process** for extracting structured ESG data from unstructured PDF and text documents.

Technical Implementation & Tools

  • Utilized **AWS Bedrock** to access pre-trained large language models (LLMs) without managing infrastructure, allowing AI-driven document analysis.
  • Implemented **Retrieval-Augmented Generation (RAG)** to combine LLM reasoning with domain-specific knowledge stored in custom knowledge bases for precise data extraction.
  • Built knowledge bases in **Amazon Bedrock** to store ESG-related data, ensuring the AI can reference historical context, financial standards, and regulatory requirements.
  • Developed **Python scripts** for document preprocessing, including PDF text extraction, cleaning, and tokenization, using libraries such as **PyPDF2**, **PDFPlumber**, and **NLTK/Spacy**.
  • Designed **ETL pipelines** using Python and Pandas to normalize, validate, and enrich extracted ESG data before storing in databases or S3 buckets.
  • Stored processed data in **Amazon S3**, with metadata maintained in **PostgreSQL** for structured queries and fast retrieval.
  • Integrated **AWS Lambda** functions with **API Gateway** to allow dynamic data extraction requests from the Angular frontend.
  • Implemented **Angular frontend** for internal users to submit requests, review extracted data, and trigger report generation.
  • Automated report generation in multiple formats (CSV, JSON, Excel) using **Python Pandas**, enabling easy downstream analytics.
  • Ensured **security and compliance** by managing sensitive credentials with **AWS Secrets Manager** and encrypting data at rest with **KMS**.
  • Deployed workflow orchestration using **AWS Step Functions**, providing structured task execution and error handling.

Business & Technical Analysis

  • Analyzed existing manual ESG extraction workflows to identify bottlenecks and error sources.
  • Collaborated with stakeholders to define business rules for ESG metrics and reporting standards.
  • Mapped document structures and financial tables to create robust parsing logic.
  • Designed a scalable architecture capable of handling large document volumes across multiple reporting periods.
  • Ensured solution was **extensible**, allowing integration of additional financial data types and new regulatory reporting requirements.

Key Achievements & Impact

  • Fully automated the extraction of ESG data from annual PDF reports, reducing processing time from days to minutes.
  • Achieved **high accuracy and consistency** in ESG data extraction by combining RAG with knowledge bases.
  • Enabled **real-time report generation** for internal and client-facing dashboards.
  • Delivered a **scalable and maintainable architecture**, supporting future expansion to additional ESG report types and other document sources.
  • Improved business decision-making by providing **clean, structured ESG data** for compliance, risk assessment, and investment analytics.
  • Documented full **technical user guide and system architecture** for developers to maintain, extend, and deploy the solution efficiently.

  • Cloud & Serverless: AWS Lambda, API Gateway, Step Functions, Secrets Manager, KMS
  • Frontend: Angular, Bootstrap, HTML/CSS
