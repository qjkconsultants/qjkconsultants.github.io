Project Overview

This project implements a modular, production-ready ETL pipeline using Apache Airflow to orchestrate extraction, transformation, and loading of data into Amazon Redshift. The pipeline automates the full lifecycle of a data warehouse, including creation and management of staging, dimension, and fact tables, ingestion of raw JSON data from S3, and comprehensive data quality validations.

As a Senior Data Engineer, I designed the pipeline to be fully scalable, maintainable, and modular, enabling other engineers to extend functionality with minimal guidance. Decisions such as templated S3 keys, parameterized JSONPaths, and configurable COPY commands reflect best practices for enterprise-grade ETL.

Technical Implementation & Tools

  • Developed ETL pipeline using Python and Apache Airflow DAGs for orchestration and scheduling.
  • Implemented custom Airflow operators to encapsulate key ETL logic, improve reusability, and enforce consistency across multiple tables:
    • StageToRedshiftOperator – loads JSON files from S3 into staging tables with templated S3 keys, prefix checks, and error handling.
    • LoadDimensionOperator – inserts data into dimension tables (users, songs, artists, time) with overwrite/append strategies, pre/post SQL support, and row-count validation.
    • LoadFactOperator – inserts data into fact tables (songplays, unmatched_songplays) with optional pre/post SQL, validation, and fail-fast behavior.
    • DataQualityOperator – executes automated data quality tests (null checks, uniqueness, referential integrity) and raises failures for operational alerting.
    • RedshiftTableCreator – modularized table drop/create tasks using SQLExecuteQueryOperator for maintainable schema management.
  • Designed DAG to support parallel staging and dimension/fact loads, reducing runtime and increasing throughput for large datasets.
  • Templated S3 keys allow dynamic execution and backfills, demonstrating forward-thinking design for production workloads.
  • Parameterization of region, JSONPaths, and compression ensures flexibility across multiple Redshift clusters and environments.
  • Integrated logging, retries, and robust error handling, ensuring operational reliability and maintainability.
  • Implemented strict data validation and referential integrity, ensuring the pipeline delivers accurate and trustworthy data for business intelligence and analytics.
  • Leveraged AWS services (Redshift, S3, IAM) following secure best practices for credentials and access management.
